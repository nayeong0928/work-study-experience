## 스프링

### 스프링

- 자바의 오픈소스 애플리케이션 프레임워크
- 객체 관리 컨테이너 제공
- 무겁고 러닝커브가 높다
- IOC, DI, AOP 지원
- POJO이기 때문에 라이브러리 기술에 종속되지 않음

### 스프링부트

- 스프링 기반 앱의 설정을 간단히 할 수 있도록 해줌
- 내장서버(톰캣) 포함하기 때문에 따로 톰캣을 설치/관리할 필요 없음
- Starter를 이용하여 의존성 자동화

## 빌드 관리 도구

### 1. Maven

- 자바용 프로젝트 관리 도구
- pom.xml을 통해 환경설정
- 고정적이고 선형적인 단계의 모듈 기반

### 2. Gradle

- Groovy 언어 사용하는 프로젝트 관리 도구
- build.gradle을 통해 환경설정
- Incremental Build: 예전 빌드 이후 변경된 이력이 없으면 태스크를 다시 수행하지 않음
- Build Cache: 다른 빌드에서 사용된 결과물 사용 가능

## 스프링의 동작 방식

### 의존성 주입

- 객체 간의 의존관계를 미리 설정해두면 스프링 컨테이너가 의존관계를 자동으로 주입
- ApplicationContext: 스프링의 조립기로써 객체 생성과 의존성 주입을 관리해준다
- 의존대상이 바뀌면 조립기만 변경하는 방식으로 의존성 주입
- 클래스 간의 의존성을 줄여 결합도를 줄여준다

### Spring MVC의 요청 처리 순서

1. 요청이 오면 DispatcherServlet이 먼저 처리
2. HandlerMapping에게 어느 컨트롤러에 보낼지 물어봄
3. 컨트롤러에서는 요청 처리할 메소드를 RequestMapping
4. Service DI 받아서 비즈니스 요청 처리
5. 이때 DB 접근이 필요하면 DAO가 위임 (DAO는 SQL쿼리로 DB에서 정보를 받아온다)
6. Service에서 로직이 끝나면 결과를 컨트롤러로 보내준다
7. 컨트롤러는 결과를 ModelAndView 형태로 DispatcherServlet으로 보낸다
8. DispatcherServlet은 ViewResolver에게 뷰 정보를 보냄
9. ViewResolver는 뷰정보로 응답할 View를 찾아서 응답
10. View에 렌더링해서 클라이언트에게 보낸다
